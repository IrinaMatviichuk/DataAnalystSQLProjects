# Revenue by Device and Continent Analysis

## Project Overview

This project focuses on analyzing revenue distribution across continents and device types using session, order, and account data.

The goal was to build an aggregated analytical dataset for comparing revenue contribution, user activity, and account verification across regions.

---

## Business Problem

Understanding where revenue comes from is essential for evaluating market performance and user behavior.

This analysis helps answer several business questions:

* Which continents generate the highest revenue?
* How much revenue comes from mobile versus desktop users?
* What share of total revenue does each continent contribute?
* How many active accounts and sessions are associated with each region?
* How many accounts are verified?

These insights can support market prioritization and device-specific business strategies.

---

## Dataset

The analysis combines data from multiple relational tables:

* `order`
* `product`
* `session_params`
* `account_session`
* `account`

The final dataset aggregates transactional, behavioral, and account-level information.

---

## SQL Approach

The query was built using several Common Table Expressions (CTEs).

### 1. Revenue Calculation

The first CTE calculates total revenue for each session by joining order data with product prices.

Revenue is calculated as:

```sql
SUM(product.price)
```

This creates a session-level revenue dataset.

---

### 2. Session Data Preparation

The second CTE prepares session-related attributes:

* session identifier
* account identifier
* device type
* continent
* account verification status

This step enriches session data with account metadata.

---

### 3. Aggregation by Continent

The third CTE aggregates all metrics at the continent level.

Conditional aggregation is used to split revenue by device:

* mobile revenue
* desktop revenue

Additional metrics include:

* account count
* verified account count
* session count

---

### 4. Revenue Share Calculation

The final query calculates each continent’s share of total revenue using a window function.

Formula:

```sql
continent revenue / total revenue * 100
```

This enables direct comparison between regions.

## Key Metrics

### Revenue

Total revenue generated by the continent.

### Revenue from Mobile

Revenue generated from mobile sessions.

### Revenue from Desktop

Revenue generated from desktop sessions.

### % Revenue from Total

Percentage contribution to total revenue.

### Account Count

Number of unique accounts associated with the continent.

### Verified Account

Number of verified accounts.

### Session Count

Number of unique sessions.
